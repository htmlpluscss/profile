eCommerce Data Extraction
API Documentation

Welcome to the eCommerce Data Extraction API documentation. This API allows you to extract categories, product details, and entire catalogs from any eCommerce store's website. The API returns the extracted data in JSON format.

Authentication

The API requires authentication using an API key. You can obtain an API key by signing up for an account on our website. Once you have an API key, you must include it in every API request you make, using the Authorization header:

Authorization: Bearer <API_KEY>

Retrieving Categories

To extract all categories from an eCommerce store, make a GET request to the API endpoint at:

GET https://api.example.com/categories?url=<STORE_URL>

The API endpoint accepts one query parameter:

  • url: the URL of the eCommerce store's website.

For example, to extract all categories from an eCommerce store with the URL https://www.example.com, you would make a GET request to:

GET https://api.example.com/categories?url=https://www.example.com

The API will return a JSON object containing an array of all the categories found on the eCommerce store's website:

{ "categories": [ "Category 1",
"Category 2",
"Category 3",
... ] }

Retrieving Product Details

To retrieve product details, make a GET request to the API endpoint at:

GET https://api.example.com/products/details?url=<PRODUCT_URL>

The API endpoint accepts one query parameter:

  • url: the URL of the product details page on the eCommerce store's website.

For example, to retrieve the details for a product with the URL https://www.example.com/product123, you would make a GET request to:

GET https://api.example.com/products/details?url=https://www.example.com/product123

Retrieving Entire Catalog (BETA)

To extract the entire catalog of an eCommerce store, make a GET request to the API endpoint at:

GET https://api.example.com/products/catalog?url=<STORE_URL>

The API endpoint accepts one query parameter:

  • url: the URL of the eCommerce store's website.

For example, to extract the entire catalog of an eCommerce store with the URL https://www.example.com, you would make a GET request to

GET https://api.example.com/products/catalog?url=https://www.example.com

Response Format

The API returns the extracted data in JSON format. The JSON object contains various product details such as the product ID, name, description, price, image URL, category, and brand, as well as all the categories extracted from the eCommerce store's website.

Summary

To use this API, you will need to make HTTP requests to the API endpoints with the appropriate parameters. Here are the endpoints for each type of data:

Categories Endpoint

  • Endpoint: /categories
  • Method: GET
  • Response Format: JSON
  • Query Parameters: none

Product Details Endpoint

  • Endpoint: /products/:product_id
  • Method: GET
  • Response Format: JSON
  • Query Parameters: product_id - the unique identifier for the product you want to retrieve details for

Entire Catalog Endpoint

  • Endpoint: /catalog
  • Method: GET
  • Response Format: JSON
  • Query Parameters: none

For example, to retrieve the details of a specific product with ID 123, you would send a GET request to /products/123. The response would be a JSON object containing all the details of the product.

Similarly, to retrieve all the categories available in the eCommerce store, you would send a GET request to /categories. The response would be a JSON object containing an array of all the categories.

Finally, to retrieve the entire catalog of products in the eCommerce store, you would send a GET request to /catalog. The response would be a JSON object containing an array of all products in the catalog and an array of all the categories.

Please note that depending on the size of the catalog and the number of categories in the store, the response may take some time to generate. However, the API is designed to process requests quickly and efficiently, so you should receive a response within a matter of seconds.

Error Handling

If an error occurs while processing an API request, the API will return an HTTP status code indicating the error, along with an error message in JSON format.
Here are some common HTTP status codes that may be returned by the API:

  • 400 Bad Request: the API request was invalid or missing required parameters.
  • 401 Unauthorized: the API request was not authenticated properly.
  • 404 Not Found: the product details could not be found at the provided URL.
  • 500 Internal Server Error: an error occurred while processing the API request.

Rate Limits

To ensure fair usage of the API, we enforce rate limits on API requests. The default rate limit is 100 requests per hour per API key. If you need a higher rate limit, please contact us to discuss your requirements.
